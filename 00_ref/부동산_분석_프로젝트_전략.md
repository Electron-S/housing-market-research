# 프로젝트 전략 — 한국 부동산 분석

## 운영 원칙

이 저장소는 한국 부동산 자산·지역·단지·개발사업지를 분석하는 전용 파이프라인이다.

핵심 구조:

1. 문서 생성 코드(`03_code/`)와 시각화 파이프라인은 분석 유형에 무관하게 재사용한다
2. 도메인 의존성은 `02_plan/`, `00_ref/` 문서에 집중 관리한다
3. 중간 산출물 구조(`04_workspace/[id]_KR`)로 분석 대상별 독립 작업 공간을 유지한다

## 전환 후 운영 모델

### 입력

- 분석 대상 ID
- 자산 종류
- 지역 범위
- 분석 목적
- 기준 시점

### 처리

1. 거시환경 정리
2. 대상 개요 작성
3. 가격/임대 흐름 정리
4. 수요·공급 분석
5. 입지 분석
6. 비교 자산 벤치마크
7. 리스크 구조화
8. 시나리오 및 전략 제안
9. 판단 트리/지표 트리 작성
10. 보고서 구성 확정
11. 본문 집필
12. 품질 리뷰
13. DOCX 변환

### 출력

- 투자 검토 보고서
- 입지 분석 보고서
- 시장 모니터링 리포트
- 개발사업 초기 검토서

## 권장 타깃 유형

| 유형 | 예시 | 핵심 지표 |
|---|---|---|
| 아파트 단지 | 잠실엘스 | 실거래가, 전세가율, 학군, 공급 |
| 오피스 | 성수동 중형 오피스 | 평당 임대료, 공실률, 흡수율 |
| 상가 | 역세권 근린상가 | 유동인구, 업종 믹스, 임대수익률 |
| 물류센터 | 수도권 동남권 | 공실률, 임대료, 접근성 |
| 개발사업지 | 재개발/재건축 후보지 | 용적률, 사업속도, 정책 리스크 |

## 문서 톤

- 중개 광고 문구 금지
- 근거 없는 상승 전망 금지
- 데이터 출처와 기준 시점 명시
- 의사결정 관점으로 결론을 명확히 제시
- "좋다/나쁘다"보다 "왜 그런지"를 구조적으로 설명

## 운영 체크리스트

- [x] 프로젝트 개요 문서 (`00_ref/`)
- [x] 워크플로우 및 템플릿 (`02_plan/`)

- [x] 평가축: 가격/수급/입지/정책/리스크 중심
