# STEP14: 품질 리뷰 (DOCX 단계)

## 목적

STEP13에서 변환된 DOCX 파일의 포맷·데이터·가독성을 최종 검증한다.
DOCX 변환 과정에서 발생할 수 있는 서식 깨짐, 표 레이아웃 문제, 데이터 누락을 잡는다.

## 입력

- `04_workspace/[target_id]_[agent]_KR/report_designed_[agent].docx`
- `STEP12_output_[agent].md` (MD 리뷰 결과 참조용)

## 검증 체크리스트

### 1. 포맷 검증

- [ ] 제목·소제목 스타일이 계층 구조에 맞게 적용됐는가
- [ ] 표 헤더 배경색·테두리선이 정상 적용됐는가
- [ ] 표 셀 내 줄바꿈이 자연스럽게 처리됐는가
- [ ] 헤더·푸터가 정상 출력되는가
- [ ] 페이지 번호가 정상 표시되는가
- [ ] 페이지 경계에서 표나 문단이 부자연스럽게 잘리지 않는가

### 2. 데이터 정확성 검증

- [ ] 주요 수치(분양가·실거래가·토지확보율 등)가 STEP9·STEP11과 일치하는가
- [ ] 출처·기준 시점 표기가 변환 후에도 유지됐는가
- [ ] 비교 단지 표의 수치가 STEP7과 일치하는가

### 3. 가독성 검증

- [ ] 한국어 줄바꿈이 자연스러운가 (단어 중간 절단 없음)
- [ ] 표 열 너비 균형이 적절한가
- [ ] Executive Summary가 1페이지 안에 들어오는가

### 4. 문자수 재확인

```bash
.venv/bin/python 03_code/count_docx_chars.py \
  04_workspace/[target_id]_[agent]_KR/report_designed_[agent].docx
```

- [ ] 8,000~12,000자 범위 유지 확인

### 5. 내용 최종 확인

- [ ] STEP12 REVISE 항목 수정 내용이 DOCX에 반영됐는가
- [ ] Executive Summary에 분양가 판단·수주 타당성 결론이 선제 배치됐는가
- [ ] STEP0 수주 전/후 분기에 맞는 결론이 명시됐는가

## 판정

| 판정 | 조건 | 다음 행동 |
|---|---|---|
| **PASS** | 모든 체크리스트 통과 | STEP15 진행 |
| **포맷 수정** | 서식·레이아웃 문제만 있음 | `improve_docx_design.py` 재실행 또는 직접 수정 후 재검증 |
| **내용 수정** | 수치 오류·누락 있음 | STEP11 복귀 → STEP12 → STEP13 재수행 |
