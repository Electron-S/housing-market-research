# STEP 1: 의뢰 정의

## 목적
분석 범위·목적·기준 시점을 확정하여 이후 모든 단계의 기준선을 만든다.
이 단계에서 정의가 모호하면 수집 데이터의 범위와 결론의 방향이 흔들린다.
사업구역이 특정된 경우에는 해당 구역에 대한 GS건설 사업 내용을 먼저 확인해 분석의 출발점을 맞춘다.

## 입력
- 의뢰자 요청 (분석 대상, 목적, 납기)
- `target_id` 결정 (예: `seongsu-residential_KR`)
- 보고서 유형 결정: `아파트 시장조사 보고서` 또는 `상가 시장조사 보고서`
- 사용자가 제시한 사업구역 명칭

## 작업 체크리스트
- [ ] 분석 목적 선택: 투자·매입·매각·개발·모니터링 중 하나
- [ ] 보고서 유형 선택: 아파트 시장조사 보고서 / 상가 시장조사 보고서
- [ ] 산출물 유형 선택: 권역 시장 브리프·사업지 입지 분석·경쟁 단지 비교 중 하나
- [ ] 기준 시점 확정 (예: 2026-04)
- [ ] 대상 범위 정의: 단일 자산 / 생활권 / 행정구역
- [ ] `01_data/[target_id]/01_gs_project/` 및 `99_notes/` 폴더 준비
- [ ] 사업구역 기준 GS건설 사업 내용 인터넷 검색 및 선행 정리
- [ ] 핵심 가설 1~3개 작성
- [ ] 산출물 유형에 따른 필수 STEP 확인 (마스터 워크플로우 참조)
